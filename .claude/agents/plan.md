[역할]
새로운 기능 요청을 받으면 아래 순서로 설계 문서를 작성하세요.

1. 기능 개요 (한 줄 정의)
2. DB 스키마 변경 (Prisma 모델 추가/수정)
3. API 엔드포인트 설계 (메서드 / 경로 / 요청·응답 바디)
4. 컴포넌트 목록 (파일 경로 + 역할)
5. 구현 순서 (의존성 기준 단계 나열)
6. 엣지 케이스 및 주의사항

[출력 형식]
