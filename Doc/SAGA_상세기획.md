# SAGA — 플랫폼 상세 기획서

> AI 소설·캐릭터 채팅 플랫폼  
> 버전 1.0 | 개인 PC 베타 기준

---

## 0. 서비스 개요

### 한 줄 정의
> "AI와 함께 소설을 쓰고, AI 캐릭터와 대화하는 창작 플랫폼"

### 핵심 가치
- **창작자**: 내가 만든 캐릭터·스토리를 AI로 생동감 있게 구현
- **독자/플레이어**: 일방적으로 읽는 소설이 아닌, 내가 참여하는 인터랙티브 스토리
- **커뮤니티**: 창작물을 공유하고 다른 사용자의 작품을 탐색

### 서비스 구조
```
SAGA
├── 스토리 탭    — AI가 함께 쓰는 인터랙티브 소설 (챕터 진행)
├── 캐릭터 탭   — AI 캐릭터와 1:1 실시간 대화 (롤플레잉)
├── 내 작품      — 내가 만든 스토리·캐릭터 관리
└── 관리자       — 전체 콘텐츠·유저 관리 (ADMIN 전용)
```

---

## 1. 사용자 유형

| 유형 | 설명 | 주요 행동 |
|------|------|-----------|
| **플레이어** | 타인 작품을 즐기는 독자 | 탐색 → 플레이 → 좋아요·북마크 |
| **크리에이터** | 스토리·캐릭터를 만드는 제작자 | 작품 생성 → 공개 → 피드백 확인 |
| **관리자** | 플랫폼 운영자 | 유저·콘텐츠 전체 관리 |

> 베타 단계에서 플레이어와 크리에이터는 동일 계정(USER)으로 운용

---

## 2. 전체 사용자 플로우

```
[미인증]
랜딩/홈 → 회원가입 또는 로그인
              ↓
[인증 후]
홈 피드 (스토리 탭)
    │
    ├── 장르 선택 → 필터링된 목록
    ├── 작품 카드 클릭 → 상세 페이지
    │       ├── 스토리 → 플레이 시작 → 챕터 진행
    │       └── 캐릭터 → 채팅 시작 → 실시간 대화
    │
    ├── 캐릭터 탭 전환 → 캐릭터 목록
    ├── 사이드바 → 최근 대화 이력
    └── 내 작품 → 생성·수정·삭제
```

### 3-1. 홈 피드 — 스토리 탭 (`/`)

#### 레이아웃
```
┌─── 헤더 (고정) ────────────────────────────────────────┐
│ SAGA  [스토리] [캐릭터]   🔍검색    🔔    [프로필]      │
├─── 장르바 (sticky) ────────────────────────────────────┤
│ [전체] [로맨스] [판타지] [현대] [무협] [SF] [공포] ...  │
├────────────┬───────────────────────────────────────────┤
│            │  ── 피처드 배너 ──────────────────────── │
│  사이드바  │  슬라이드 배너 (3~5개, 5초 자동 전환)      │
│            │                                           │
│  ─ 최근   │  ── 섹션: 지금 인기 ──────────────────── │
│  대화      │  [카드][카드][카드][카드][카드] → (가로스크롤)│
│            │                                           │
│  ─ 북마크  │  ── 섹션: 새로 올라왔어요 ────────────── │
│            │  [카드][카드][카드][카드][카드]             │
│            │                                           │
│            │  ── 섹션: 이 장르 어때요? (로맨스) ─────  │
│            │  [카드][카드][카드][카드]                  │
└────────────┴───────────────────────────────────────────┘
```

#### 피처드 배너 상세
- 배너 1개: 가로 전체 너비, 높이 360px
- 배경: 커버 이미지 + 좌측 그라데이션 오버레이
- 요소: 장르 뱃지 + 제목(H1) + 설명 2줄 + "시작하기" CTA 버튼
- 우측 하단: `1/5` 형식 카운터
- 자동 슬라이드 5초, 마우스 호버 시 일시 정지

#### 스토리 카드 규격
```
┌──────────────────────┐  ← 전체: 164 × 335px 고정
│  [커버 이미지]        │  이미지 영역: full width × 247px
│                      │  상단: 장르 뱃지 (로맨스, 판타지 등)
│                      │
├──────────────────────┤
│ 제목 (최대 2줄)       │  텍스트 영역: 88px (335 - 247)
│ 작가명                │
│ ♥ 1.2k  👁 5.6k       │
└──────────────────────┘
```

#### 섹션 구성 (장르별 다름)
| 섹션명 | 정렬 기준 | 카드 수 | 표시 방식 |
|--------|-----------|---------|-----------|
| 지금 인기 | 좋아요 + 플레이 수 복합 | 전체 조회 | flex-wrap 그리드 (최대 6슬롯/행) |
| 새로 올라왔어요 | 등록일 최신순 | — | **임시 숨김 처리 (향후 복원 예정)** |
| 이 장르 어때요? | 해당 장르 인기순 | 10개 | 가로 스크롤 |
| 완결된 작품 | status=COMPLETED | 10개 | 가로 스크롤 |

### 3-2. 홈 피드 — 캐릭터 탭 (`/character`)

#### 레이아웃 차이점 (스토리 탭과 다른 부분)
- 배너: 3열 세로형 그리드 (aspect-ratio 0.6)
- 트렌드 랭킹 번호(1~N) 배너 좌상단 표시
- 카드: 세로형 (portrait), 대화 수 표시

#### 캐릭터 배너 (상단 3열 그리드)
```
┌──────┐ ┌──────┐ ┌──────┐
│  01  │ │  02  │ │  03  │  ← 랭킹 번호
│      │ │      │ │      │  세로형 이미지
│      │ │      │ │      │
├──────┤ ├──────┤ ├──────┤
│이름  │ │이름  │ │이름  │
│1줄설명│ │1줄설명│ │1줄설명│
│💬 2만 │ │💬 1만 │ │💬 8천 │
└──────┘ └──────┘ └──────┘
```

#### 섹션 구성
| 섹션명 | 내용 |
|--------|------|
| 요즘 뜨는 캐릭터 | 최근 7일 채팅 증가율 기준 |
| 새로운 캐릭터 | 등록일 최신순 |
| 장르별 인기 | 각 장르 상위 4개씩 |
| 캐릭터 모아보기 | 정렬 필터 포함 전체 목록 |

### 3-5. 스토리 플레이 (`/story/[id]/play`)

#### 뷰어 레이아웃
- 뷰어 전체 너비: **900px 고정**
- 정렬: **가운데 정렬** (사이드바 우측 공간 기준)
- 상태 패널 열릴 때: `marginRight: 256px` 적용하여 좌측으로 이동
- 내부 콘텐츠: 부모 900px 컨테이너가 너비를 담당 (`max-w-2xl mx-auto` 미적용)

#### 화면 구성
```
┌───────────────────────────────────────────────────────┐
│  ← 뒤로   스토리명 — 3화: 위험한 밤                   │
├───────────────────────────────────────────────────────┤
│                                                       │
│   **빗소리가 창문을 두드리는 밤, 이준이 갑자기         │
│   방문을 열었다.**                                     │
│                                                       │
│   "왜 연락이 없었어."                                  │
│                                                       │
│   *채은은 심장이 쿵 내려앉는 걸 느꼈다.*               │
│                                                       │
├───────────────────────────────────────────────────────┤
│  선택지 (추천 응답):                                   │
│  [ "미안해요, 바빴어요."  ]                            │
│  [ "당신이 올 줄 몰랐어요." ]                          │
│  [ "왜 갑자기요?"          ]                          │
├───────────────────────────────────────────────────────┤
│  직접 입력:  [__________________] [전송 ▶]            │
│              🔄 자동재생   ✏️ 수정   🌿 분기           │
└───────────────────────────────────────────────────────┘
```

#### 핵심 기능
| 기능 | 설명 |
|------|------|
| **선택지** | AI가 추천 3개 생성, 클릭 시 해당 내용으로 전송 |
| **직접 입력** | 자유 텍스트 입력 |
| **자동재생** | AI가 사용자 행동까지 생성하며 자율 전개 |
| **수정** | 마지막 AI 응답 재생성 요청 |
| **분기** | 현재 시점에서 새 채팅방 생성 (다른 선택 탐색) |
| **메모리 보기** | 장기기억 목록 열람·편집 (우측 서랍) |

#### 텍스트 출력 포맷
- **행동·묘사**: `**별표**` 사이에 표시, 색상 구분(회색 계열)
- **대화**: 큰따옴표, 흰색 텍스트
- **내면**: *이탤릭*, 투명도 낮춤
- **상태창(HUB)**: 코드블록 형태, 항상 응답 하단

### 3-8. 작품 생성 — 스토리 (`/story/create`)

#### 단계별 입력 (스텝 UI)
```
스텝 1/4  기본 정보
├── 제목 (필수, 최대 50자)
├── 설명 (필수, 최대 500자)
├── 장르 (다중 선택, 최대 3개)
└── 태그 (선택, 최대 10개)

스텝 2/4  커버 이미지
├── 이미지 업로드 (직접)
└── AI 생성 (프롬프트 입력 → SD WebUI)

스텝 3/4  첫 번째 챕터
├── 챕터 제목
├── 시작 씬 (오프닝 상황 설명)
└── AI 생성 모드 (기본/롤플레잉/시뮬레이션)

스텝 4/4  공개 설정
├── 공개 여부 (전체공개 / 링크공개 / 비공개)
└── 완결 여부 (연재중 / 완결)
```

### 3-9. 작품 생성 — 캐릭터 (`/character/create`)

#### 단계별 입력
```
스텝 1/4  기본 정보
├── 이름 (필수)
├── 한 줄 소개 (필수, 최대 100자)
├── 상세 설명 (최대 1000자)
```

## 4. 기능 명세

### 4-1. 인증

| 항목 | 명세 |
|------|------|
| 회원가입 | loginId(4~20자 영문숫자) + password(6자 이상) + nickname(2~15자) |
| 이메일 | 불필요, 수집 안 함 |
| 비밀번호 | bcrypt(10) 해시 저장 |
| 중복 검사 | loginId, nickname 각각 실시간 체크 (onBlur) |
| 로그인 | ID/PW 입력 → NextAuth Credentials → JWT 발급 |
| 세션 | JWT, 기본 30일 유지 |
| 로그아웃 | 세션 무효화, 홈으로 이동 |
| 초기 관리자 | `admin / 1234` (Seed 자동 생성) |

### 4-1-1. 스토리 API 스펙

| 항목 | 명세 |
|------|------|
| `GET /api/stories` | 스토리 목록 조회 |
| `limit` 파라미터 미전달 또는 `0` | `take` 없이 **전체 조회** |
| `limit=N` 명시 | N개 반환 (상한 제한 없음) |
| 정렬 파라미터 | `orderBy` 쿼리로 인기순·최신순 전환 가능 |

> 변경 이력: 기존에는 `limit` 상한 50개를 강제하였으나, 홈 피드 그리드 전체 표시를 위해 제한 해제 (2026-05-18)

### 4-2. 스토리 플레이

| 항목 | 명세 |
|------|------|
| 플레이 세션 | `PlaySession` 모델로 관리. 사용자 × 스토리 조합으로 세션 생성 |
| 플레이 메시지 | `PlayMessage` 모델에 저장. `role`: USER / ASSISTANT |
| 진행 저장 | `PlaySession.chapterId` + `turnCount`로 진행 위치 추적 |
| 캐릭터 상태 | `PlaySession.charStatus` 필드에 JSON 직렬화하여 저장 |
| 이어하기 | 마지막 완료 챕터 다음부터 시작 |
| 분기 | 현재 메시지 기준으로 새 ChatSession 생성 (향후 구현 예정) |
| 자동재생 | 사용자 행동 → 캐릭터 응답을 AI가 연속 생성 (향후 구현 예정) |
| 수정 | 마지막 ASSISTANT 메시지 삭제 → 재생성 (향후 구현 예정) |
| 선택지 | 직전 AI 응답 기반 3개 추천, 클릭 시 자동 전송 (향후 구현 예정) |

### 4-3. 캐릭터 채팅

| 항목 | 명세 |
|------|------|
| 세션 | 사용자 × 캐릭터 = 1개 세션 (`ChatSession` 모델, 재입장 시 이어서 대화) |
| 스트리밍 | Socket.IO `/character-chat` 네임스페이스, chunk 이벤트 |
| 타이핑 표시 | AI 생성 중 `⠶ 입력 중...` 표시 |
| 메모리 | 슬라이딩 윈도우 방식 (`CONTEXT_WINDOW_SIZE = 20`) |
| 초기화 | 채팅 기록 전체 삭제 (확인 모달) |
| 히스토리 제한 | 최근 20개 메시지만 컨텍스트에 포함 (`lib/memory.ts`) |
| 메모리 4요소 | 장기/단기/관계도/목표 DB 저장은 **향후 구현 예정** |

### 4-4. 좋아요 / 북마크

## 9. 기술 아키텍처 (기술구조 분석 기반)

> 참조 플랫폼 역분석 결과를 SAGA 구현에 맞게 적용한 아키텍처 설계

### 9-1. 서비스 전체 흐름

```
┌──────────────────────────────────────────────────────────┐
│                사용자 (브라우저)                           │
└───────────────────────┬──────────────────────────────────┘
                        │ HTTPS
┌───────────────────────▼──────────────────────────────────┐
│           Next.js :3000 (SSR + API Routes)                │
│   - 페이지 SSR (getServerSideProps 유사)                  │
│   - REST API Routes (/api/*)                              │
│   - 정적 파일 (/public/uploads/*)                         │
└────────┬──────────────────────────┬──────────────────────┘
         │ SWR fetch                │ Socket.IO
┌────────▼──────────┐   ┌──────────▼──────────────────────┐
│  SQLite / PG DB   │   │   Socket Server :3001             │
│  (Prisma ORM)     │   │   /character-chat 네임스페이스    │
└───────────────────┘   └─────────────────────────────────┘
         │
┌────────▼──────────────────────────────────────────────────┐
│  Claude CLI (child_process)  →  Anthropic 클라우드        │
│  SD WebUI :7860              →  이미지 생성 (선택)         │
└───────────────────────────────────────────────────────────┘
```

### 9-2. API 서비스 분류

참조 플랫폼의 마이크로서비스 구조를 SAGA의 Next.js API Routes로 매핑합니다.

| 참조 플랫폼 서비스 | SAGA 대응 경로 | 담당 기능 |
|------------------|---------------|-----------|
| `/auth/` | `/api/auth/` | 인증·세션 |
| `/crack-api/` | `/api/stories/`, `/api/characters/` | 콘텐츠 CRUD |
| `/crack-gen/` | `/api/chat/sessions/`, `/api/play-sessions/`, `/api/stories/[id]/play-message` | AI 채팅·플레이 세션 |
| `/crack-page/` | `/api/stories/` (쿼리 파라미터 필터링) | 홈 피드·장르 데이터 |
| 키워드북 | `/api/characters/[id]/keywords/` | 동적 컨텍스트 삽입 |
| SSE 이벤트 | `/api/generate/text` (SSE) | 스트리밍 텍스트 (**향후 구현 예정**) |
| Socket.IO | `socket-server/:3001` | 실시간 채팅 (`socket-server/index.ts` 단일 파일) |
| 이미지 생성 | `/api/generate/image` | SD WebUI 연동 |

|------|------|------|
| `sessionId` | String | 사용자×캐릭터 고유 세션 |
| `role` | Enum | USER / ASSISTANT |
| `content` | String | 메시지 원문 |

### 9-4. 실시간 통신 설계

두 채널을 병용합니다.

| 채널 | 기술 | 용도 |
|------|------|------|
| **텍스트 스트리밍** | SSE (`/api/generate/text`) | 스토리 챕터 생성, AI 응답 청크 전송 |
| **캐릭터 채팅** | Socket.IO (`/character-chat`) | 실시간 1:1 대화, 타이핑 표시, 채팅 동기화 |

**스트림 딜레이** (참조 플랫폼 분석 결과 적용)
- 자연스러운 타이핑 효과를 위해 청크 출력에 인위적 딜레이 적용
- 권장값: `delayRatio 1.8 / maxDuration 500ms / slice 5`

```typescript
// 타이핑 효과 구현 예시
async function* streamWithDelay(generator: AsyncGenerator<string>) {
  for await (const chunk of generator) {
    yield chunk;
    await new Promise(r => setTimeout(r, 30)); // 청크당 30ms 딜레이
  }
}
```

## 10. AI 글쓰기 시스템 상세 (시스템프롬프트 분석 기반)

> 실제 AI 채팅·스토리 생성이 어떻게 동작하는지 구현 수준으로 정의

### 10-1. 컨텍스트 레이어 구조 (L1~L6)

LLM에 전달되는 프롬프트는 6개 레이어가 순서대로 쌓입니다.
우선순위는 L1이 가장 높고 아래로 갈수록 낮아집니다.

```
[L1] 플랫폼 시스템  ← 절대 고정 (안전 필터, 출력 형식, 한국어 강제)
[L2] 캐릭터 설정    ← 제작자 입력 (성격·말투·세계관·행동 규칙)
[L3] 메모리         ← 장기기억 / 단기기억 / 관계도 / 목표
[L4] 사용자 레이어  ← 대화 프로필 (이름·나이·성별) + 유저노트
[L5] 대화 기록      ← 최근 20턴 원문
[L6] 출력 제어      ← HUB 위치, 최대 토큰, 포맷 강제
```

### 10-2. L1 — 플랫폼 고정 지시 (변경 불가)

```
모든 응답은 반드시 아래 규칙을 따르세요.

[출력 형식]
1. 한국어 웹소설 문체로 작성
2. 행동·묘사 → **별표 두 개** 사이
   예: **그가 창밖을 바라보며 한숨을 내쉬었다.**
3. 대화 → 큰따옴표
   예: "괜찮아. 걱정 마."
4. 내면 묘사 → *이탤릭* 또는 별도 단락
5. 내부 분석 과정(Chain-of-Thought) 절대 출력 금지
6. 외국어 혼용 금지
```

> **알려진 한계**: L1 지시가 있어도 LLM 특성상 완전 제어 불가
> - 외국어 혼재: 의미 벡터 공간에서 언어 경계 불명확 → 한국어 강제 지시로 최소화
> - CoT 누출: 추론 과정이 응답에 혼입될 수 있음 → 사용자에게 [재생성] 버튼 제공

### 10-3. L2 — 5종 캐릭터 템플릿

제작자가 캐릭터 생성 시 선택하는 AI 행동 모드입니다.

| 템플릿 | AI 행동 | 적합 장르 |
|--------|---------|-----------|
| **기본** | 범용, 폭넓은 상황 이해 | 일상·힐링 |
| **1:1 롤플레잉** | 캐릭터 고정, 1인칭 감정 반응 중심 | 로맨스·판타지 |
| **시뮬레이션** | 3인칭 GM, 다중 NPC·분기 서사 | RPG·어드벤처 |
| **생산성** | 전문가 모드, 소설 문체 해제 | 정보·학습 |
| **커스텀** | 제작자가 raw 프롬프트 직접 입력 | 고급 사용자 |

### 10-4. L2 — 캐릭터 설정 구조

제작자가 입력한 내용이 L2 프롬프트로 변환되는 구조입니다.

```
# 캐릭터: [이름]
성격: [키워드들]
말투: [예: 짧은 반말, '~야', 고압적]
외모: [외모 묘사]
직업/역할: [역할]
현재 상황: [시작 설정]

# 세계관
배경: [시대/장소/규칙]
장르: [로맨스판타지/현판/시뮬레이션 등]

# 행동 규칙
- 절대 캐릭터를 벗어난 발언 금지
- 사용자를 항상 [이름]으로 부를 것
- [제작자 추가 규칙...]

↓ [AI 자동 완성]
위 설정을 기반으로 캐릭터 성격·말투를 구체화하는 디스크립션 자동 추가
```

### 10-5. L3 — 메모리 4요소

| 요소 | 설명 | 업데이트 | 편집 |
|------|------|---------|------|
| **장기기억** | 영구 보존 과거 사건 | 단기→장기 자동 이동 | ✅ 사용자 직접 편집 (최대 100개) |
| **단기기억** | 최근 대화 흐름 타임라인 요약 | 매 대화 후 자동 갱신 | ❌ 자동 관리 |
| **관계도** | 캐릭터↔사용자 현재 관계·감정 | 매 대화 후 자동 갱신 | ❌ 자동 관리 |
| **목표** | 현재 스토리 목표 | 자동 갱신 | ❌ 자동 관리 |

**메모리 전환 흐름**
```
대화 N턴 진행
    ↓
단기기억 생성 (최근 흐름 1~2문장 요약)
    ↓
중요 사건 감지 시 → 장기기억으로 이동
    ↓
관계도·목표 자동 업데이트
    ↓
다음 대화 = L2(캐릭터) + L3(메모리 4요소) + 최근 20턴
```

### 10-6. L4 — 사용자 레이어

| 항목 | 설명 | 입력 위치 |
|------|------|-----------|
| **대화 프로필** | 스토리 내 사용자 이름·나이·성별·외형 | 채팅 설정 |
| **유저노트** | 캐릭터에게 기억시킬 자유 입력 (최대 2000자) | 채팅 설정 |

**유저노트 활용 예시**
```
"나는 고소공포증이 있어."
"HUB는 반드시 답변 최하단에만 출력할 것."
"다음 대화부터 3일 후 상황으로 진행해줘."
```

> L4는 L2(제작자 설정)보다 우선순위가 낮음 → L1 안전 정책 위반 지시는 무시됨

### 10-7. 상태창 HUB/HUD 시스템

RPG의 HUD처럼 날짜·장소·수치를 코드블록으로 출력하는 기능입니다.

**출력 형식**
````
```현재 상태
📅 날짜: 3월 9일 | 일요일 | 맑음
📍 장소: 학교 옥상
💗 호감도: ████░░ 68%
🎯 회차: #145
```
````

**핵심 규칙 — HUB는 반드시 응답 하단에 배치**
- 상단 배치 시 AI가 "매번 같은 방식으로 시작" 신호로 인식 → **동일 답변 반복 버그 발생**
- L6에서 `"HUB는 반드시 응답 최하단에만 출력할 것"` 강제
- 유저노트에도 동일 지시 권장

### 10-8. 키워드북 (동적 컨텍스트 삽입)

대화 중 특정 키워드 감지 시 관련 정보를 L2에 자동 삽입하는 시스템입니다.

```
트리거 키워드: "마왕성"
감지 시 삽입 내용: "마왕성은 북쪽 산악지대의 검은 성. 수천 년 역사..."

→ 사용자가 "마왕성 얘기 해줘" 입력 시
→ 위 내용이 L2 뒤에 자동으로 추가되어 AI가 참조
```

**활용 예시**
| 키워드 | 삽입 목적 |
|--------|-----------|
| `!요약` | AI가 현재 서사를 요약하도록 트리거 |
| `마왕성` | 세계관 상세 정보 삽입 |
| `❤️` | 호감도 시스템 트리거 |
| 캐릭터 이름 | 해당 NPC 상세 설정 삽입 |

### 10-9. 분기 (Branch) 기능

특정 대화 시점에서 새로운 세션을 생성해 다른 방향의 스토리를 탐색합니다.

```
현재 대화 흐름:
[대화 1] → [대화 2] → [대화 3] → [분기 생성]
                                      ↓
                              [새 세션 A] → 다른 선택으로 전개
                              (기존 세션 유지)
```

**구현 방식**
- 특정 메시지에서 "분기" 버튼 클릭
- 해당 시점까지의 메시지를 복사한 새 `ChatSession` 생성
- L2(캐릭터 설정)·L3(메모리)는 분기 시점 상태 그대로 복사
- 이후 대화만 다르게 전개

### 10-10. 자동재생 (Auto-Play) 기능

사용자 입력 없이 AI가 사용자 행동과 캐릭터 응답을 모두 생성해 이야기를 자율 전개합니다.

```
일반 대화: [사용자 입력] → [AI 응답] → 반복
자동재생:
  Step 1: AI가 "사용자의 다음 자연스러운 행동" 생성
  Step 2: 생성된 행동 기반으로 "캐릭터 응답" 생성
  Step 3: 반복 (기본 3턴)
```

**UI**: 자동재생 중 일시정지 버튼 표시, 완료 후 계속/중단 선택

### 10-11. 알려진 AI 동작 이슈 및 대응

| 이슈 | 원인 | SAGA 대응 방법 |
|------|------|---------------|
| **동일 답변 반복** | HUB 상단 배치로 토큰 선택 범위 제한 | HUB 기본값 하단, L6·유저노트에 강제 지시 |
| **CoT 누출** | AI 추론 과정이 응답에 혼입 | L1에 `"분석 과정 출력 금지"` + [재생성] 버튼 제공 |
| **외국어 혼재** | LLM 의미 벡터 공간에서 언어 경계 불명확 | L1 한국어 강제 + 이후 최적화 |
| **맥락 고착** | 직전 대화 패턴 반복으로 전개 고착 | 단기기억 매 턴 갱신, 사용자 유저노트로 돌파 가능 |
| **캐릭터 이탈** | 복잡한 지시 충돌 시 페르소나 해제 | L2 첫 줄에 `"당신은 [이름]입니다"` 항상 삽입 |
