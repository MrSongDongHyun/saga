## 1. WRTN 스토리 빌더 전체 구조 분석

crack.wrtn.ai의 스토리 만들기는 **8개 탭**으로 구성된 단계별 빌더다.  
각 탭의 핵심 기능과 SAGA 적용 방향을 정리한다.

## 2. 탭별 상세 분석

### Tab 1 — 프로필 (Profile)

| 필드 | 규격 | 설명 |
|------|------|------|
| 이미지 | 5MB 이하, 1,080×1,620px (세로형) | 업로드 / 삭제 / AI 생성 |
| 이름 | 2~30자, 특수문자·이모지 제외 (필수) | 스토리 제목 |
| 한 줄 소개 | 0/30자 (필수) | 짧은 태그라인 |
| 랜덤 생성 | 버튼 1개 | 이름+소개 자동 생성 |

**SAGA 현황:** 이미지/제목/설명 있음. **한 줄 소개(태그라인)** 없음.  
**추가 필요:** `tagline` 필드(DB), 1,080×1,620 비율 이미지 권장

### Tab 2 — 스토리 설정 (Story Settings)

#### 2-1. 프롬프트 템플릿 (필수)
5가지 사전 정의 템플릿:

| 코드 | 이름 | 특징 |
|------|------|------|
| `basic` | 기본 프롬프트 | AI가 다양한 상황을 폭넓게 이해 |
| `roleplay` | 1:1 롤플레이 프롬프트 | AI가 사람처럼 감정 표현 |
| `simulation` | 시뮬레이션 프롬프트 | 게임·멀티 캐릭터 상황에 강함 |
| `productivity` | 생산성 프롬프트 | 전문 지식 기반 답변 |
| `custom` | 제작자 커스텀 프롬프트 | 직접 시스템 프롬프트 전체 작성 |

#### 2-2. 스토리 설정 및 정보 (필수, 0/4000자)
- 세계관, 설정, 등장인물 외모/성격/말투 등 상세 정보
- **자동 생성** 버튼 (AI가 초안 작성)

#### 2-3. 전개 예시 (Few-shot Examples, 최대 3개)
- 유저 입력(0/500) + AI 출력(0/500) 쌍
- **전체 자동 생성** / 예시 추가 버튼
- AI 응답 품질을 높이는 Few-shot 프롬프팅

#### 2-4. 우측 채팅 미리보기
- 설정 입력 중 실시간 채팅 미리보기 패널
- 모델 선택 드롭다운 (슈퍼펫 2.0)
- 첫 메시지 직접 테스트 가능

**SAGA 현황:** 스토리 설명 필드 있으나 단순 textarea.  
**추가 필요:** 프롬프트 템플릿 선택 UI, 전개 예시 등록, 우측 채팅 미리보기

### Tab 3 — 시작 설정 (Start Settings)

복수의 시작 설정 지원 (기본 설정 + 추가 가능).  
각 시작 설정별로 완전히 다른 스토리 분기 구현 가능.

| 필드 | 규격 | 설명 |
|------|------|------|
| 시작설정 이름 | 0/12자 (필수) | 예: "기본 설정", "어둠의 분기" |
| 프롤로그 | 0/1000자 (필수) | 플레이 시작 시 유저에게 보이는 오프닝 |
| 시작 상황 | 0/1000자 | AI에게 전달되는 초기 컨텍스트 (유저 역할, 관계, 세계관) |
| 플레이 가이드 | 0/500자 | AI가 기억 안 함. 유저에게만 보이는 플레이 안내 |
| 추천 답변 | 최대 3개 | 첫 턴 추천 선택지 |

**SAGA 현황:** 세력/배경 선택 있음. 프롤로그/시작상황 없음.  
**추가 필요:** 프롤로그 표시 UI, 복수 시작 설정, 플레이 가이드 (유저 전용 안내)

### Tab 4 — 스탯 설정 (Stat Settings)

시작 설정별 최대 **7개** 커스텀 스탯 정의.  
현재 SAGA의 무협 고정 STATUS를 **제네릭 커스텀 스탯 시스템**으로 전환.

#### 스탯 필드 구조

| 필드 | 규격 | 설명 |
|------|------|------|
| 스탯 이름 | 0/10자 (필수) | 예: 전투력, 호감도, 인지도 |
| 아이콘 | 15종 선택 (필수) | ❤️⭐⚫🕐👾🔑✏️🛡️💜👤⚠️🚩🔮 등 |
| 색상 | 9색 선택 (필수) | 주황/노랑/청록/초록/하늘/파랑/핑크/보라/빨강 |
| 기본값 | min / max / default | 범위: -99,999 ~ 99,999 |
| 단위 | 0/3자 | %, 개, 순위 등 |
| 스탯 설명 | 0/500자 (필수) | AI가 언제 스탯을 올리고 내릴지 판단 기준 |
| 자동생성 | 버튼 | AI가 스탯 설명 초안 작성 |

#### 스탯 레벨 설정
- 값 범위별로 레벨명 + 프롬프트(0/100자) 지정
- 레벨에 따라 AI 응답 방식이 달라짐
- 예: 전투력 100~500 = "초보", 500~2000 = "중급" 등

**SAGA 현황:** 무협 전용 하드코딩 STATUS (경지/내공/명성 등).  
**전환 필요:** 완전 커스텀 스탯 DB 모델 설계 및 UI 재구성

### Tab 5 — 미디어 (Media)

AI가 대화 상황을 분석하여 **자동으로 적절한 이미지 선택** 표시.

#### 이미지 업로드 규칙
- 파일명 규칙: `분류_상황.jpg`  
  예) `에피_기름.jpg` → 분류: 에피 / 상황: 기름
- 1회 최대 1,000장, 1장당 5MB 이하
- 적용 범위: 전체 / 특정 시작 설정

#### 이미지 배치표 (Image Placement Matrix)
- 우측 패널에 분류×상황 매트릭스 표시
- 예: 분류 1 · 상황 2 · 총 3/80 (등록된 / 가능한 슬롯)
- 검색으로 분류/상황 필터 가능

#### 이미지 생성 버튼
- AI 이미지 생성 (SD WebUI와 연동 가능)

**SAGA 현황:** SD WebUI 연동 이미지 생성 있음.  
**추가 필요:** 상황 기반 이미지 자동 선택 로직, 파일명 컨벤션 처리, 배치표 UI

### Tab 7 — 엔딩 설정 (Ending Settings)

SAGA에 없는 **핵심 게임 시스템**. 스탯 조건 충족 시 특별 엔딩 카드 제공.

#### 엔딩 등급 (4단계)
| 등급 | 희귀도 | 의미 |
|------|--------|------|
| N | 노말 | 일반 엔딩 |
| R | 레어 | 특별 루트 |
| SR | 슈퍼 레어 | 숨겨진 엔딩 |
| SSR | 슈퍼 슈퍼 레어 | 최고 등급 엔딩 |

#### 엔딩 필드 구조

| 필드 | 규격 | 설명 |
|------|------|------|
| 엔딩 이미지 | 5MB, 1,005×1,490px (필수) | 카드 이미지, AI 생성 가능 |
| 엔딩 등급 | N/R/SR/SSR (필수) | 희귀도 |
| 엔딩 이름 | 0/20자 (필수) | |
| 엔딩 조건 | 스탯 + 연산자 + 값 (필수) | 최소 10턴, 5턴마다 체크 |
| 엔딩 가능 시점 | 턴 수 지정 | X턴 이후부터 조건 체크 |
| 프롬프트 | 0/500자 (필수) | AI 엔딩 판단 상세 조건 설명 |
| 에필로그 | 0/1000자 | 엔딩 후 서사, 자동 생성 가능 |
| 엔딩 힌트 | 0/20자 | 유저에게 보이는 힌트 |

#### 엔딩 조건 연산자
`>` `≥` `<` `≤` `=` `≠` — 6가지 비교 연산자  
단일 규칙 + 규칙 그룹(AND/OR) 조합, 최대 7개 규칙

**SAGA 현황:** 엔딩 시스템 없음.  
**신규 구현 필요:** 완전 신규 기능

## 4. 개발 단계별 계획

### Phase 1 — 스토리 설정 고도화 (핵심 기능)
**목표:** WRTN 수준의 스토리 작성 UX 달성

#### Task 27 — 프롬프트 템플릿 + 스토리 설정 고도화
- DB: `Story` 모델에 `promptTemplate` (enum), `storyInfo` (text, 4000자), `exampleDialogs` (JSON) 추가
- UI: 스토리 편집 페이지에 템플릿 선택 드롭다운

- 시스템 프롬프트: storyInfo + exampleDialogs를 play-stream에 반영

#### Task 28 — 프롤로그 + 시작 상황 컨텍스트 추가
- DB: `StorySession`에 `prologue` (text), `startContext` (text) 추가

- 플레이 페이지: 첫 턴에 프롤로그 자동 표시

### Phase 2 — 커스텀 스탯 시스템 (핵심 기능)
**목표:** 무협 하드코딩 STATUS → 완전 자유 커스텀 스탯

#### Task 29 — 커스텀 스탯 DB 모델 설계
```prisma
model StoryStatDef {
  id          String   @id @default(cuid())
  storyId     String
  story       Story    @relation(fields: [storyId], references: [id])
  name        String   // 스탯 이름 (최대 10자)
  icon        String   // 아이콘 코드
  color       String   // 색상 코드
  unit        String?  // 단위 (%, 개 등)
  minVal      Int      @default(0)
  maxVal      Int      @default(100)
  defaultVal  Int      @default(50)
  description String   // AI 판단 기준 설명
  order       Int      // 표시 순서
  levels      StatLevel[]
}

model StatLevel {
  id        String       @id @default(cuid())
  statDefId String
  statDef   StoryStatDef @relation(fields: [statDefId], references: [id])
  name      String       // 레벨명
  minVal    Int
  maxVal    Int
  prompt    String?      // 레벨별 AI 지시 (100자)
}

model PlayStatValue {
  id          String       @id @default(cuid())
  sessionId   String
  statDefId   String
  value       Int
  updatedAt   DateTime     @updatedAt
}
```

#### Task 30 — 커스텀 스탯 편집 UI
- 스토리 편집 페이지에 스탯 설정 탭 추가
- 스탯 추가/삭제/순서 변경
- 아이콘 선택 (15종), 색상 선택 (9색)
- min/max/default 값 입력
- 레벨 설정 (범위 + 이름 + 프롬프트)

#### Task 31 — play-stream에 커스텀 스탯 반영
- 시스템 프롬프트 동적 생성 (고정 무협 STATUS → 커스텀 스탯 정의 주입)
- STATUS_UPDATE 파서를 커스텀 스탯 키로 변환
- 플레이 페이지 STATUS 패널 → 커스텀 스탯 렌더링

### Phase 3 — 엔딩 시스템 (핵심 기능)
**목표:** N/R/SR/SSR 등급 엔딩 카드 + 스탯 조건 체크

#### Task 32 — 엔딩 DB 모델 설계
```prisma
model StoryEnding {
  id         String          @id @default(cuid())
  storyId    String
  story      Story           @relation(fields: [storyId], references: [id])
  grade      EndingGrade     // N | R | SR | SSR
  name       String          // 엔딩 이름 (20자)
  image      String?         // 카드 이미지 URL
  prompt     String          // AI 판단 조건 (500자)
  epilogue   String?         // 에필로그 (1000자)
  hint       String?         // 유저 힌트 (20자)
  minTurn    Int             @default(10)
  startTurn  Int             @default(10)
  conditions EndingCondition[]
}

model EndingCondition {
  id        String       @id @default(cuid())
  endingId  String
  ending    StoryEnding  @relation(fields: [endingId], references: [id])
  statDefId String
  operator  String       // > >= < <= = !=
  value     Int
  groupId   Int?         // 그룹 OR 조건용
}

enum EndingGrade { N R SR SSR }
```
